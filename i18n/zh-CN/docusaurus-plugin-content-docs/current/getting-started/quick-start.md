# 快速开始

只需 5 分钟即可上手 OpsKat。本指南将带你完成初始设置、添加第一台服务器以及执行第一次 AI 辅助操作。

## 首次启动

从应用菜单启动 OpsKat，或直接运行构建好的二进制文件。首次启动时，应用会自动初始化 SQLite 数据库并创建数据目录。

主界面左侧边栏包含：
- **资产** — 你的基础设施清单
- **AI 智能体** — 自然语言运维操作
- **设置** — 配置与偏好设置

## 添加第一个资产

### SSH 服务器

1. 点击资产侧边栏中的 **+** 按钮。
2. 选择 **SSH** 作为资产类型。
3. 填写连接信息：
   - **名称** — 显示名称（例如"生产环境 Web 服务器"）
   - **主机** — 主机名或 IP 地址
   - **端口** — SSH 端口（默认：22）
   - **用户名** — SSH 登录用户
   - **认证方式** — 选择 `密码` 或 `密钥`
   - **密码 / 私钥** — 你的凭据
4. 点击 **保存**。

凭据在存储到本地数据库之前，会使用 AES-256-GCM 进行加密。加密主密钥通过操作系统密钥环管理。

### 数据库（MySQL / PostgreSQL）

1. 点击 **+** 并选择 **数据库**。
2. 选择**驱动** — MySQL 或 PostgreSQL。
3. 输入主机、端口、用户名、密码和默认数据库名。
4. 可选择一个 SSH 资产作为隧道连接。
5. 点击 **保存**。

### Redis

1. 点击 **+** 并选择 **Redis**。
2. 输入主机、端口和可选的密码。
3. 可选择一个 SSH 资产作为隧道连接。
4. 点击 **保存**。

## 连接 SSH 终端

1. 从侧边栏选择一个 SSH 资产。
2. 点击 **连接** 或双击该资产。
3. 一个交互式终端会话将在新标签页中打开。

你可以对终端进行分屏、打开多个会话，还可以使用内置的 SFTP 文件浏览器传输文件。

## 配置 AI 智能体

AI 智能体允许你通过自然语言管理基础设施。使用前需要先配置提供商。

### 方式一：OpenAI 兼容 API（推荐）

1. 打开 **AI 智能体** 标签页 — 首次使用时会出现设置向导。
2. 选择 **OpenAI 兼容** 作为提供商类型。
3. 输入 API 端点和 API 密钥。
4. 选择模型（点击**获取模型**自动发现，或手动输入 — 模型参数会自动填充）。
5. 点击 **完成**。

支持任何 OpenAI 兼容的 API，包括 DeepSeek、Azure OpenAI、自托管模型等。

### 方式二：Anthropic API

1. 在设置向导中，选择 **Anthropic** 作为提供商类型。
2. 输入 API 密钥。
3. 选择 Claude 模型（例如 `claude-sonnet-4-20250514`）。
4. 点击 **完成**。

你可以在 **设置 > AI** 中管理多个提供商并随时切换。

## 你的第一次 AI 对话

1. 打开 **AI 智能体** 标签页。
2. 输入自然语言指令，例如：

   > 查看我的生产服务器的磁盘使用情况

3. AI 将会：
   - 调用 `list_assets` 查找可用的服务器
   - 调用 `run_command` 在目标资产上执行 `df -h`
   - 以易读的格式返回结果

4. 如果命令匹配了**拒绝**策略，将被阻止。如果需要确认，执行前会提示你进行审批。

所有操作都记录在[审计日志](/docs/guide/audit)中，包含完整的决策追踪。
