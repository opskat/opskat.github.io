# RDP 远程桌面

OpsKat 内置 RDP 客户端，用于连接 Windows 服务器和桌面。会话直接打开在 OpsKat 工作区中，可与终端和数据面板并行使用，无需再启动独立的远程桌面应用。

## 创建 RDP 资产

添加资产并选择 **RDP**，然后配置：

- **主机 / 端口** —— 远程 Windows 主机与 RDP 端口（默认 `3389`）。
- **用户名 / 密码** —— Windows 登录凭据；可存在资产中，也可选择已托管的凭据。
- **域** —— 可选的 Active Directory 域。
- **剪贴板** —— 启用剪贴板同步，默认开启。
- **连接方式** —— 支持直连、SSH 隧道或 SOCKS 代理；SSH 隧道与代理二选一。

保存前可用 **测试连接** 验证端点和凭据。

## 使用远程桌面

双击资产或选择 **连接** 即可打开 RDP 页面。工具栏提供：

- **适应** 和 **实际尺寸** 两种显示模式。
- **全屏** 模式。
- **Ctrl+Alt+Del**、**Alt+Tab** 和 **Esc** 特殊按键。
- 会话期间剪贴板开关。
- 断开和重连控件。

鼠标移动、按键、滚轮、键盘扫描码、Unicode 文本、远程光标形状和画面更新都在内嵌会话中处理。

## 剪贴板与文件

启用剪贴板同步后：

- 本地复制的文本可粘贴到远程桌面，远程文本也可复制回本地。
- 本地复制的文件可通过 RDP 剪贴板文件传输提供给远程桌面。
- 远程桌面复制的文件会先接收到本地临时目录，再暴露给操作系统剪贴板。

剪贴板会跨越信任边界。连接不可信主机时，建议在工具栏或资产设置中关闭。

## 连接说明

- **直连** 由运行 OpsKat 的机器直接连接 RDP 端点。
- **SSH 隧道** 通过所选 SSH 资产转发 RDP TCP 连接。
- **SOCKS 代理** 通过配置的代理路由连接。
- 远程桌面尺寸会跟随当前面板，资产中保存的尺寸用作默认值。

RDP 目前是交互式桌面功能，不定义放行/拦截策略类型，也不提供专用的 `opsctl rdp` 命令。
